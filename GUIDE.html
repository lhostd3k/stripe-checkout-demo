🛒 Stripe Checkout Demo

Complete guide — from zero to deployed. University assignment · $1.00 test payment · Node.js + Express + Stripe

1 · Architecture

This app has three parts that work together:

Browser (Frontend)HTML page with "Buy" button
Calls your server via fetch()
⟷
Your Server (Backend)Node.js + Express
Creates Stripe session
Your Server (Backend)Returns Stripe URL
⟷
Stripe (External)Hosts checkout page
Processes payment

Step-by-step flow

1
User opens yoursite.com — sees homepage with the "Buy $1 Test Item" button
2
User clicks the button → browser sends POST /create-checkout-session to your Express server
3
Your server calls the Stripe API using your secret key, creating a Checkout Session for $1.00
4
Stripe returns a unique one-time URL like https://checkout.stripe.com/c/pay/cs_test_...
5
Your server sends that URL back to the browser. The browser redirects to Stripe's hosted page.
6
User enters card details on Stripe's secure page (you never touch card data)
7
Stripe redirects to /success.html (paid) or /cancel.html (cancelled)

⚠️ Why use a backend? Your Stripe secret key must never be in the browser. The backend keeps it safe on the server and acts as the middleman.

2 · File Structure

stripe-checkout/ ← root project folder ├── public/ ← static files served to browser │ ├── index.html ← homepage with Buy button │ ├── success.html ← shown after successful payment │ └── cancel.html ← shown if user cancels ├── server.js ← Node.js + Express backend ├── package.json ← project dependencies ├── .env ← YOUR SECRET KEYS (never commit!) ├── .env.example ← template showing what goes in .env └── .gitignore ← tells Git to ignore .env and node_modules

The public/ folder is served automatically by Express as static files. The browser can request any file inside it directly.

3 · Code (Each File)

All code files are included as separate files in this project. Below is a quick reference for what each file does.

Frontend

public/index.html — Homepage

The main page users see. Contains the large "Buy $1 Test Item" button and a product card showing the item name, description, and $1.00 price. When clicked, calls startCheckout() which fetches /create-checkout-session and redirects to Stripe.

Frontend

public/success.html — Payment Confirmed

Shown after successful payment. Displays a receipt card with item name, amount ($1.00), and a "Paid" status badge. Includes a back button to the store.

Frontend

public/cancel.html — Payment Cancelled

Shown when user clicks "back" or closes Stripe checkout. Explains no charge was made and offers a "Try Again" button back to the homepage.

Backend

server.js — Express Server

The heart of the app. Serves static files from /public and handles one API route:

  • POST /create-checkout-session — creates a Stripe Session for $1.00 (100 cents) and returns { url: "https://checkout.stripe.com/..." }
  • GET /health — simple health check (useful for Render.com)
Config

package.json

Lists the three dependencies: express (web server), stripe (Stripe SDK), and dotenv (loads .env file). Also sets "start": "node server.js" which Render.com uses.

Secret — Never Commit

.env (you create this)

STRIPE_SECRET_KEY=sk_test_YOUR_KEY_HERE
YOUR_DOMAIN=http://localhost:3000
PORT=3000

Copy .env.example to .env and fill in your real values. This file is in .gitignore so it is never uploaded to GitHub.

4 · Stripe Setup Instructions

Step 1 — Create a Stripe account

1
Go to https://stripe.com and click Start now
2
Enter your email, name, country, and create a password
3
Verify your email address
4
You do NOT need to enter bank details to use test mode

Step 2 — Get your API keys

1
Log in to the Stripe Dashboard at https://dashboard.stripe.com
2
Make sure Test mode is ON — look for the toggle in the top right. It should say "Test mode" and the keys should start with sk_test_ and pk_test_
3
Click Developers → API keys
4
Copy the Secret key (starts with sk_test_). Click "Reveal" if it's hidden.
5
Paste it into your .env file as STRIPE_SECRET_KEY=sk_test_...

⚠️ Test mode vs Live mode: Always use test mode for this assignment. Test keys start with sk_test_. Live keys start with sk_live_ — never use those for demos.

5 · Local Development Setup

Prerequisites

You need Node.js installed. Check by running:

node --version   # should print v18.x.x or higher
npm --version    # should print 9.x.x or higher

If not installed, download from https://nodejs.org (choose the LTS version).

Install and run locally

# 1. Enter the project folder
cd stripe-checkout

# 2. Install dependencies (creates node_modules/ folder)
npm install

# 3. Create your .env file from the example
cp .env.example .env

# 4. Open .env in any text editor and fill in your Stripe secret key
#    STRIPE_SECRET_KEY=sk_test_YOUR_REAL_KEY_HERE
#    YOUR_DOMAIN=http://localhost:3000

# 5. Start the server
npm start

# You should see:
#   ✅ Server running at http://localhost:3000

Open http://localhost:3000 in your browser. You should see the homepage.

8 · Debugging & Common Errors

❌ Button does nothing / page stays the same

Cause: Server isn't running, or browser can't reach it.

Fix: Open browser DevTools (F12) → Console tab. Look for red errors. Make sure npm start is running and the terminal shows no errors.

❌ "Server configuration error: missing Stripe secret key"

Cause: STRIPE_SECRET_KEY is not set in your environment.

Local fix: Make sure .env file exists and contains STRIPE_SECRET_KEY=sk_test_...

Render fix: Go to your service → Environment → verify the variable is added and saved, then redeploy.

❌ "Server configuration error: missing YOUR_DOMAIN"

Cause: YOUR_DOMAIN env var is missing.

Fix: Add it to .env (local) or Render environment variables. Make sure there's no trailing slash.

❌ Checkout opens but redirects to localhost after payment

Cause: YOUR_DOMAIN is still set to http://localhost:3000 on Render.

Fix: Update YOUR_DOMAIN in Render's environment variables to your actual Render URL (e.g. https://my-app.onrender.com).

❌ "No such API key" or "Invalid API key provided"

Cause: The secret key is wrong, has extra spaces, or you copied the publishable key instead of the secret key.

Fix: Secret keys start with sk_test_. Publishable keys start with pk_test_ — those won't work here. Go to Stripe Dashboard → Developers → API Keys and re-copy.

❌ Render says "Build failed"

Fix: Check your Render deploy logs. The most common causes are: missing package.json, wrong start command, or a syntax error in server.js. Make sure package.json has "start": "node server.js".

❌ App is slow to load on Render (free tier)

Cause: Free Render services "spin down" after 15 minutes of inactivity. The first request after that takes 30–60 seconds to wake up.

Fix: Just wait for it to wake up. Once running, it responds instantly.

❌ CORS error in console

Cause: This shouldn't happen with this setup, but if it does it means the frontend and backend are on different domains.

Fix: Both frontend and backend must be served from the same Express server. Don't deploy them separately.
